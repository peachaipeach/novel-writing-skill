# 写小说 (Novel Writing)

## 核心指导原则

### 底层原则：人物是故事的本体（v3.0 新增）

> **在思考"这章要推进什么"之前，先问"这个人物此刻正在经历什么"。**
> 情节是人物选择的结果，而非人物为情节服务的工具。

1. **人物不是"承载剧情的容器"** — 角色不是为了让剧情发生而存在的，剧情是因为"他是这样的人"才自然发生的
2. **从"需要发生什么"转向"这个人在此情此景会做什么"** — 相信你的人物，让他们自己做出选择
3. **三要素（爽点/代入感/期待感）是设计工具，写作时不要想着它们** — 设计阶段用三要素规划，写作时放下三要素，只想着"这个人在此刻的真实体验"
4. **鲜活的人物本身就是最大的爽点和代入感来源** — 一个读者真正关心的人，做什么读者都想看
5. **相关技法见 **`references/character-vitality.md`**（人物鲜活度）和 `references/immersive-narrative.md`（沉浸式叙事）**

### 写作三要素（始终牢记）

1. **爽点** — 读者读到的"爽"感，剧情的高潮与回报，让读者获得情感满足
2. **代入感** — 读者能否代入角色、沉浸故事，产生"我就是主角"的感觉
   - **主角不能过于完美**，应有普通人的缺点和犹豫，让读者觉得"他和我一样"
   - **主角动机应贴近现实**：如"保护家人""活下去"等基本诉求，让读者能理解和共情
   - **通过主角视角叙事**：描述其所见、所感、所想，让读者产生"如果我在那个世界，我也会这么做"的感觉
3. **期待感** — 读者对后续发展的期待与好奇，驱动读者继续读下去
   - **设置悬念**：通过未解之谜和未知信息勾起好奇心
   - **明确目标**：给主角清晰的目标，让读者有盼头
   - **制造危机**：通过危机感和紧迫感迫使读者想知道"后来呢"

### 写作风格理念

1. **环境描写可以有，但不能因字数限制减少人物心理描写**
2. **人物互动是故事的核心**，应该多写人物之间的交流与碰撞
3. **环境描写服务于人物和情节**，不能喧宾夺主
4. **画面感 = 情绪编织，而非表情堆积** — 不靠堆砌人物有多少种表情，而靠编织背后隐藏的情绪
5. **有灵性的对话** — 避免单调的"某某说""某某道"，用行动、语气、上下文来传递谁在说话
6. **句式服务于情绪** — 紧张时多用短句，抒情时多用长句

### 内容安全规范

1. **都市题材**：不使用现实中真实存在的城市名、地名等，使用虚构地名
2. **历史与现实人物**：不能抹黑正面历史人物或现实人物
3. **社会描写**：不能过度展示社会黑暗面
4. **平等原则**：不能有任何性别或种族歧视内容
5. **宗教敏感**：涉及宗教信仰的地方要谨慎处理

## 启动流程（每次触发时执行）

skill 被触发后，按以下流程与用户互动。

### Step 1：初次沟通 — 了解用户需求

先打招呼，然后快速了解用户当前在做什么：

> "你好！很高兴一起创作故事 🎉
>
> 你目前处于哪个阶段？
> 1. **新项目** — 想写一个全新的故事（创意构思 / 搭建大纲）
> 2. **已有草稿** — 已有写好的章节，需要继续写或修改
> 3. **已完成初稿** — 写完了第一版，需要编辑润色
> 4. **有问题想问** — 写作中遇到具体问题（人物怎么设计、某个场景怎么写等）
>
> 另外，你的小说是什么类型？（玄幻 / 都市 / 仙侠 / 科幻 / 悬疑 / 言情 / 历史 / 其他）"

根据用户的选择，进入对应的模式（见 Step 2）。

**路径记录**：同时询问并记录以下路径以备后用：
- 输入目录（已有草稿/参考资料的位置，可选，默认当前目录）
- 输出目录（生成文件的存放位置，可选，默认 `./小说/`）

### Step 2：进入对应模式

#### 模式 A：新项目 · 创意构思

**目标**：帮助用户从零到一构思故事的核心创意

**执行流程**：

1️⃣ **理解用户的初始想法**
   - "有没有大致的故事方向？或者只是一个模糊的感觉？"
   - 如果没有任何想法 → 用 `references/creative-process.md` 中的"头脑风暴方法"引导
   - 如果已经有初步方向 → 帮助提炼和深化

2️⃣ **三要素核心创意设计**（逐项引导，每项给示例）
   - **爽点**："这个故事最爽的地方在哪里？如果只能用一句话吸引读者，你会说什么？"
   - **代入感**："主角是谁？读者为什么会关心他/她？"
   - **期待感**："开篇埋什么钩子，让读者想一直看下去？"
   - 参考 `references/creative-process.md` 中的「围绕三要素设计核心创意」

3️⃣ **一句话梗概**
   - 帮助用户把故事提炼为一句能打动人的话
   - 格式："当 [激励事件] 发生，[主角] 必须 [核心行动]，否则 [危机后果]"
   - 示例："当高三学渣意外获得读心术，他必须在高考前找出隐藏在班级里的间谍，否则全校都会被毁灭。"

4️⃣ **输出**
   - 将核心创意整理成结构化的创意文档
   - 输出格式见下方「输出规范」
   - 询问用户是否要进入大纲搭建阶段

---

#### 模式 B：新项目 · 大纲搭建

**目标**：从核心创意出发，构建完整的大纲

**前提**：用户已有核心创意（可以是刚在模式 A 中构思的，也可以是用户自己已有的）

**执行流程**：

1️⃣ **角色设计**
   - 先设计主角：性格（含缺点）、动机、外貌、背景（参考 `references/creative-process.md` 中的「代入感导向的角色设计」）
   - > **核心提醒**：主角必须有缺点，不能完美；动机要接地气，不能空洞
   - 再设计重要配角（2-4 个）：与主角的关系、各自的目标
   - 设计反派：立场、与主角的核心冲突
   - **输出**：按 `references/creative-process.md` 的标准填写角色设定表

2️⃣ **世界观搭建**（视题材而定）
   - 玄幻/仙侠：力量体系、势力分布、历史传说
   - 都市：时代背景、社会阶层（注意虚构地名）
   - 科幻：科技设定、世界规则
   - 参考 `references/creative-process.md` 中的「世界观构建」和「世界观展现技法」
   - 提醒"冰山理论"：作者知道 100%，只需要展示 10%

3️⃣ **三线并行结构**
   - 确定主线、支线、暗线各是什么
   - 参考 `references/plot-structure.md` 中的「三线并行结构」
   - 主线占比 ~60%，支线 ~25%，暗线 ~15%

4️⃣ **情节节点设计**
   - 标记六大关键节点所在的章节范围
   - 参考 `references/plot-structure.md` 中的「情节节点设计」
   - 激励事件 → 第一转折点 → 中点转折 → 困境低谷 → 最终高潮 → 结局
   - 在每个节点标注三要素（这是爽点还是强化代入感？还是埋期待？）

5️⃣ **爽点分布规划**
   - 三章一小爽，十章一大爽
   - 填入 `examples/outline-template.md` 中的「爽点分布总览」

6️⃣ **逐章大纲**（最少前 10 章，建议全书）
   - 每章一行：章节号、标题、该章推进的线、本章爽点、结尾钩子
   - 参考 `references/plot-structure.md` 中的「章节结尾钩子」选择钩子类型
   - **每章结尾必须留悬念**，这是硬性要求
   - 输出格式见下方「输出规范」

7️⃣ **章节结尾钩子设计**（每一章都要有！）
   - 让用户为每章选择一个钩子类型：事件钩子 / 对话钩子 / 揭示钩子 / 危机钩子 / 信息差钩子
   - 参考 `references/plot-structure.md` 中的「章节结尾钩子」

**质量关卡**：进入下一步前检查：
- [ ] 主角是否有让人共鸣的缺点？
- [ ] 三线是否清晰区分？
- [ ] 前 10 章每章都有结尾钩子？
- [ ] 爽点分布是否三章一小爽、十章一大爽？
- [ ] 所有地名是否已虚构化（都市题材）？

#### 模式 C：已有草稿 · 续写 / 修改

**目标**：帮助用户在已有内容基础上继续写作或修改段落

**执行流程**：

1️⃣ **读取并理解**
   - 读取用户已有草稿文件
   - 快速理解：故事类型、当前进度、主要角色、核心冲突
   - 用三要素框架快速评估当前内容

2️⃣ **确定任务**
   - 询问用户具体需求：
     - "你想让我帮你续写下一章，还是修改某一段落？"
     - "续写的话，下一章你希望发生什么？（随便说个方向也行）"
     - "修改的话，具体是哪方面的问题？（节奏/对话/代入感/爽点/其他）"

3️⃣ **续写模式**
   - 先回顾前一章结尾的钩子，确保续写自然衔接
   - 检查当前章节的大纲规划（如有），按大纲走
   - 如果用户没有大纲 → 先简要规划本章：推进哪条线、核心冲突、结尾钩子
   - **写作前确认三要素**：
     - "这一章让读者爽的点是什么？"
     - "怎么让读者代入主角的处境？"
     - "结尾留什么钩子让读者想看下一章？"
   - 参考 `references/writing-craft.md` 中的对应技法
   - 写完后做三要素自检（见「质量关卡」）

4️⃣ **修改模式**
   - 让用户提供具体要修改的段落或指出问题
   - 分析问题根因（是节奏问题？对话问题？代入感问题？）
   - 给出 2-3 个修改方案供用户选择
   - 修改后对比，让用户感受差异
   - 参考对应参考文件的技法章节

5️⃣ **输出**
   - 续写：输出新的章节文件
   - 修改：输出修改后的段落附修改说明

#### 模式 D：章节精写（当用户说"帮我写第 X 章"时）

**目标**：精写单个章节，确保最高质量

**执行流程**（7 步法）：

```
步骤 1：规划 → 明确本章目的/推进哪条线/核心冲突
步骤 2：三要素确认 → 本章爽点/代入感设计/期待感+结尾钩子
步骤 3：起草 → 写出完整章节初稿
步骤 4：三要素自检 → 围绕 checklist 检查（见质量关卡）
步骤 5：打磨 → 优化对话/节奏/心理描写
步骤 6：安全审查 → 内容安全规范检查
步骤 7：交付 → 输出成文件并告知三要素评分
```

**每一步的说明**：

**① 规划（先和用户对齐）**
   - "这一章主要推进什么？（主线/支线/暗线）"
   - "核心冲突是什么？"
   - 如果用户不明确 → 根据上下文和上一章结尾自然推导
   - **关键**：不要跳过这一步直接写

**② 三要素确认**
   - 向用户确认（或根据已有信息推导）：
   - "这一章的爽点是什么？"（打脸/情感/揭示/力量爆发？）
   - "怎么让读者代入主角？"（他的感受、纠结、选择）
   - "结尾钩子选哪个类型？"（让用户选或帮选）
   - 参考 `references/plot-structure.md` 的「章节结尾钩子」

**③ 起草**
   - 根据确认的三要素进行写作
   - 注意：**以主角视角叙事**，多用心理描写，少用旁白
   - 参考 `references/writing-craft.md` 相关技法
   - 如涉及打斗 → 参考 `references/fight-scenes.md`
   - 如涉及读者心理技巧 → 参考 `references/psychology-techniques.md`

**④ 三要素自检**
   - 写完后用「质量关卡」中的三要素 checklist 自检
   - 如果有任何一项不通过 → 修改后再交付

**⑤ 打磨**
   - 检查对话是否自然（有灵性？行动代替标签？）
   - 检查句式节奏是否匹配情绪
   - 检查心理描写是否到位
   - 参考 `references/writing-craft.md`

**⑥ 安全审查**
   - 虚构地名检查（都市题材）
   - 社会黑暗面是否过度？
   - 是否有歧视性内容？
   - 参考 `references/content-guidelines.md`

**⑦ 交付**
   - 输出章节文件
   - 附简要说明：本章字数、三要素评分（★★★★★）、与大纲的匹配度

#### 模式 E：已完成初稿 · 编辑修改

**目标**：帮助用户编辑和润色已完成的小说初稿

**执行流程**：

1️⃣ **全篇阅读与评估**
   - 读取全篇，用三要素框架做整体评估
   - 给出总体评价（优缺点各 2-3 点）

2️⃣ **三轮编辑**
   - **大修（结构层面）**：参考 `references/self-editing.md`
     - 检查三幕结构、三线并行、情节逻辑、人物弧光
     - 输出：结构评估报告 + 修改建议
   - **中修（章节层面）**：参考 `references/self-editing.md`
     - 检查章节钩子、爽点分布、代入感、视角一致性
     - 逐一章节给出评价
   - **小修（文字层面）**：参考 `references/self-editing.md`
     - 精简冗余、优化句式、对话标签检查
     - 输出修改示例（找 3-5 个代表性段落做修改演示）

3️⃣ **三要素自检**（用质量关卡中的 checklist 逐一核对）

4️⃣ **内容安全审查**
   - 逐一核对 `references/content-guidelines.md` 的安全 checklist

5️⃣ **交付**
   - 编辑报告 + 修改后的文件
   - 建议优先级排序（哪些先改，哪些可以后改）

#### 模式 F：问答模式（用户有具体问题）

**目标**：回答用户的具体写作问题

**问题分类与处理路径**：

| 问题类型 | 处理方式 | 参考文件 |
|----------|----------|----------|
| "这个情节怎么设计？" | 引导三要素分析 | `plot-structure.md` |
| "这个人物怎么塑造？" | 代入感 + 不完美原则 | `creative-process.md` |
| "这段对话怎么写？" | 行动代替标签 + 潜台词 | `writing-craft.md` |
| "这场打斗怎么写？" | 感官+节奏+结果三要素 | `fight-scenes.md` |
| "怎么让读者上瘾？" | 钩子模型四步法 | `psychology-techniques.md` |
| "怎么修改我的文章？" | 大→中→小修流程 | `self-editing.md` |
| "这样写安全吗？" | 内容安全规范 | `content-guidelines.md` |
| "有没有大纲模板？" | 提供模板 | `examples/outline-template.md` |

**处理原则**：
- **先给结论，再给解释**：先直接回答用户的问题，再展开说明原理
- **每个回答配示例**：至少给一个 ✅ 好的示例和 ❌ 不好的示例
- **必要时反问**：如果用户描述太模糊，追问具体场景和需求
- **不强行套用所有三要素**：用户问一个具体问题，就针对那个问题回答，不要每次都说一遍爽点代入感期待感

#### 模式 G：写作诊断模式（v3.0 新增）

**目标**：当用户反馈"人物僵硬""像在陈述""没代入感"时，诊断根因并给出针对性改进方案

**适用场景**：
- "我感觉主角很僵硬，不鲜活"
- "写出来像是在陈述事情，没有画面感"
- "读者说看不下去，说没有代入感"
- "我不知道我的问题出在哪里"

**诊断流程**：

1️⃣ **定位具体问题**
   - 要求用户提供一段具体文本（300-500 字即可）
   - 如果不愿提供，用几个问题了解症状

2️⃣ **五维度诊断法**

   | 诊断维度 | 核心问题 | 对应参考 |
   |----------|----------|----------|
   | **人物有身体吗？** | 角色是否有触觉/温度/重量/不自觉的生理反应？ | `character-vitality.md` 身体是情绪的仪表盘 |
   | **叙事距离合适吗？** | 是在"叙述事实"还是在"让读者亲历"？ | `immersive-narrative.md` 叙事距离 |
   | **有意图裂缝吗？** | 角色想做的和他实际做的之间是否有差距？ | `character-vitality.md` 意图与行动的裂缝 |
   | **说得太多了吗？** | 是否把所有心理活动都直接写出来了？是否有留给读者的留白？ | `immersive-narrative.md` 信息滴灌与隐藏 |
   | **互动够厚吗？（v3.1）** | 关键互动有几个来回？是否跳过"发展阶段"直接到了结果？ | `scene-craft.md` 三回合原则 & 场景节奏 |

3️⃣ **给出修改方案**
   - 针对诊断出的问题，选择 1-2 个最核心的改良点
   - 给出具体的修改示例（修改前 vs 修改后）
   - 不要一次给太多建议——选最重要的 1-2 点

4️⃣ **诊断完成后**
   - 共性问题直接回答 → 模式 G 流程结束
   - 如需大幅修改 → 转入模式 C（已有草稿·修改）或模式 E（已完成初稿·编辑修改）
   - 如需学习基础技法 → 引导至对应的参考文件

**诊断示例**

> **用户**："我觉得我写的角色很僵硬，像纸片人"
>
> **诊断响应**：
> "好，给我一段你自己写的文字（300字左右），我帮你诊断具体问题。
>
> 如果暂时没有，我们先看一个最常见的僵硬原因——**角色没有身体**。
>
> 读读你刚刚写的这段话：
> 『他很紧张地走进教室，发现大家都在看他。』
>
> 问题在哪？——'紧张'是个标签，读者看不到紧张。
> ✅ 改成：
> 『他推开教室门。门发出吱呀一声。所有人的目光都转向他。他僵在门口，喉咙发紧，不知道该迈哪只脚。』
>
> **你的角色有身体吗？他感觉到了什么？**
> 可以参考 `references/character-vitality.md` 中的身体情绪对照表和微观行为词典。"

## 章节写作工作流（快速参考）

当用户说"帮我写一章"时，执行这个精简工作流：

```
① 读前一章结尾（理解当前状态）
② 问/推本章三要素（爽点是什么？钩子是什么？）
③ 沉浸（进入角色——闭上眼睛，他在哪里？周围什么声音？他的身体什么感觉？）
④ 写初稿（从人物体验出发，以主角视角，心理描写为主）
⑤ 自检（三要素 checklist + 人物鲜活度 checklist）
⑥ 交付 + 附三要素评分
```

> **关于第③步"沉浸"（v3.0 新增）**：
> 这是本次升级的关键一步。在动笔之前，不要直接想"这章要写什么"，而是先问自己：
> - "这个角色此刻的身体是什么感觉？"
> - "他在看什么、听什么、闻到什么？"
> - "他想做什么——以及他为什么没做？"
> 
> 花 10 秒钟进入角色再动笔，写出来的文字会有本质的差异。
> 参考 `references/character-vitality.md` 的「当下体验写作法」。
>
> > **关于第③步到第④步之间：互动回合数检查（v3.1 新增）**
> > 在进入角色后、动笔之前，再加一个问题：
> > - **"这场戏最关键的那场互动，我需要几个来回？"**
> > 
> > 如果你的直觉答案是"1-2个回合就够"——再想想。
> > 有意义的互动至少需要3个来回才能展开。
> > 参考 `references/scene-craft.md` 的「三回合原则」。

**核心原则**：
- 不要跳过②直接写
- 不要在写作时想着三要素——设计阶段想清楚，写作时只想着"这个人在此刻的经历"
- 每一章都要让用户知道"为什么这章好看"（三要素说明）
- 章节之间要自然衔接（从上一章的钩子出发）

## 质量关卡

### 三要素自检清单（每次交付前检查）

#### 爽点自检
- [ ] 这一章最"爽"的地方在哪里？明确吗？
- [ ] 爽点到来的节奏是否恰当？（铺垫是否够？）
- [ ] 压抑的深度是否足够支撑反弹的强度？
- [ ] 爽点是否太过重复？（同一种爽点用多了会疲劳）

#### 代入感自检
- [ ] 读者在这一章是否一直在主角视角里？
- [ ] 主角的反应是否合理？像一个普通人会做出的反应吗？
- [ ] 是否有破坏代入感的"作者旁白"？（"事实上""值得一提的是"）
- [ ] 读者会不会觉得"如果是我，我也会这么做"？

#### 期待感自检
- [ ] 这一章结束时读者是否想知道"后来呢"？
- [ ] 是否有新的悬念或目标埋下？
- [ ] 长期目标和短期目标是否都在推进？
- [ ] 危机感是否在持续？

### ★ 人物鲜活度自检清单（v3.0 新增）

> 如果读者反馈"人物僵硬""像在陈述"，优先检查这一项。参考 `references/character-vitality.md`。

#### 身体存在感
- [ ] 这一章里角色有"身体"吗？（是否提到了他的身体感受：温度/触觉/疲劳/呼吸/心跳？）
- [ ] 角色是否只存在于对话和行动中，而没有物理存在感？
- [ ] 如果删掉这个角色，场景里会不会感觉"空"？（角色的身体占据了空间）

#### 不自知的真实
- [ ] 角色是否有自己都没意识到的反应？（嘴上说不在乎但手在抖？）
- [ ] 角色嘴上说的和正在做的之间有差距吗？（意图与行动的裂缝）
- [ ] 有没有一个时刻，读者比角色自己更了解他？

#### 独特性
- [ ] 这个角色可以被换到另一本书里吗？（如果可以——他还不够独特）
- [ ] 去掉名字后，读者还能从对话中分辨出是谁在说话吗？
- [ ] 他的反应方式是"这个角色特有的"还是"放之四海而皆准的"？

#### 叙事距离（参考 `references/immersive-narrative.md`）
- [ ] 这一段是在"叙述发生了什么"还是在"让读者体验正在发生什么"？
- [ ] 关键场景的叙事距离够近吗？（摄像机在角色身边还是在天花板上？）
- [ ] 时间流速是否均匀？（如果整章节奏都一样——那大概率像在陈述）

### 内容安全自检
- [ ] 全部地名是否已虚构化（都市题材）？
- [ ] 是否有任何可能被视为丑化历史人物的内容？
- [ ] 社会黑暗面是否在合理范围内？
- [ ] 所有角色设定是否贯彻了平等原则？
- [ ] 宗教相关内容是否经过慎重处理？

### 章节验收标准（交付前逐项确认）
- [ ] 字数：2000-5000 字（网文标准，视需求可调）
- [ ] 视角：全章保持在主角视角内
- [ ] 结尾钩子：有，且明确
- [ ] 心理描写：至少 2 处（直接或间接）
- [ ] 对话：避免单调的"某某说"，有行动标签或上下文辨识
- [ ] 节奏：紧张场景短句，抒情场景长句
- [ ] 安全：通过内容安全自检

### ★ 场景互动密度自检清单（v3.1 新增）

> 如果读者反馈"互动少""一笔带过"，优先检查这一项。参考 `references/scene-craft.md`。

#### 互动厚度
- [ ] 这一章最关键的一场互动有几个来回？（≥3回合才算展开）
- [ ] 每个来回是否推进了"信息/情绪/关系"中的至少一个维度？
- [ ] 有没有"第一轮把所有底牌出了"？（情绪从0直接到10，缺少爬升过程）

#### 回合质量
- [ ] 每个对话回合是否包含台词以外的东西？（身体反应/动作/沉默/留白）
- [ ] 角色的身体反应和台词是一致的还是矛盾的？（矛盾更有戏）

#### 场景完整性
- [ ] 场景是"开场→发展→转折→收尾"四步都走完了吗？
- [ ] 有没有跳过"发展"直接到结果？（开场→直接结果=少了中间最重要的部分）
- [ ] 场景结束时角色的状态和开始时不同吗？

## 写作进度管理（v3.1 新增）

### 每五万字总结制度

> **每完成约 5 万字创作后，停下来进行阶段性总结，再继续往下写。**

这是防止"写歪了"或"写到后面忘了前面"的关键机制。5 万字大约相当于一本 10-15 万字的书的一半，或 20 万字长篇的四分之一。

#### 总结流程

当累计创作达到 5 万字时，执行以下步骤：

**第一步：通读已写内容**
- 快速通读从上次总结到现在的全部章节（重点章节精读）
- 带着问题读：人物有没有偏离设定？节奏有没有出问题？伏笔有没有忘记？

**第二步：系统性自检**

| 检查项 | 具体问题 | 参考文件 |
|--------|----------|----------|
| **人物一致性** | 主角的性格和动机是否前后一致？有没有"为了剧情而做出不符合人物的事情"？ | `character-vitality.md` |
| **互动密度** | 关键场景的互动是否连续？有没有"一笔带过"本该展开的地方？ | `scene-craft.md` |
| **爽点节奏** | 爽点间距是否合理？最近 5 万字有没有太久没有爽点？ | `plot-structure.md` |
| **伏笔回收** | 前面埋的伏笔有记着回收吗？有没有漏掉的？ | `plot-structure.md` |
| **三要素检查** | 代入感、期待感是否在线？ | 本文件质量关卡 |
| **内容安全** | 有没有违反内容安全规范的地方？ | `content-guidelines.md` |

**第三步：写总结笔记**
- 记录以下内容：
  - 这 5 万字里写得最好的 3 个场景（为什么好）
  - 这 5 万字里最不满意的 2 个地方（怎么改进）
  - 下一阶段的核心目标（接下来 5 万字要重点解决什么）
  - 有没有任何需要回头修改的地方

**第四步：调整后继续**
- 如果有需要修的地方 → 先修再继续
- 如果整体满意 → 带着下一阶段的目标继续写作
- **不要跳过这一步直接往下写**

#### 总结笔记示例格式

```markdown
## 写作阶段总结（第X个5万字）

### 📊 基础数据
- 本阶段字数：____字
- 累计总字数：____字
- 本阶段覆盖章节：第____章 ~ 第____章

### 📝 写得最好的 3 个场景
1. 第X章 [场景名] — [为什么好]
2. 第X章 [场景名] — [为什么好]
3. 第X章 [场景名] — [为什么好]

### 🔧 需要改进的 2 个地方
1. [问题 + 改进方案]
2. [问题 + 改进方案]

### 🎯 下一阶段目标
- [目标 1]
- [目标 2]

### 📌 需要回头修改的内容
- [ ] [需要修改的地方]（优先级：高/中/低）
- [ ] [需要修改的地方]（优先级：高/中/低）
```

#### 为什么要做这件事

| 不做总结的风险 | 做总结的好处 |
|---------------|-------------|
| 写到后面人物走形了 | 及时发现并纠正人物偏差 |
| 伏笔埋了忘了收 | 检查并标记未回收的伏笔 |
| 节奏越写越偏 | 重新对齐三要素和爽点节奏 |
| 前面不喜欢的部分越积越多 | 在酿成大问题前及时修 |
| 写了就忘，"回头看才发现问题" | 主动发现问题，而不是被动等待 |

## 写作场景专项指南

### 网文（连载向）
- 节奏要快，前 3 章必须有第一个小爽点
- 每章结尾的钩子要强（读者想点"下一章"）
- 爽点密度更高：可能每章都有小爽点
- 少环境描写，多对话和行动
- 字数：2000-3000 字/章

### 传统小说/出版向
- 可以更多环境描写和心理深度
- 节奏可以稍慢，但三要素同样适用
- 注重文字的质感和文学性
- 章与章之间的结构更讲究
- 字数：3000-8000 字/章

### 短篇
- 开篇第一句就必须抓住读者
- 人物少（1-3 个），冲突集中
- 爽点单一但强烈
- 结尾要有余韵（峰终定律）
- 字数：3000-15000 字

### 童话/幻想
- 代入感来自"如果我是这个孩子/动物"
- 爽点更简单直接
- 语言适合目标读者年龄
- 主题积极正面

## 对话示例

### 示例 1：用户要求写一个章节

> **用户**：帮我写第三章，主角该第一次展现实力了
>
> **Claude** 应该这样回应：
>
> "好的！写第三章前，我们先对齐一下这章的三要素：
>
> **★ 爽点**：主角第一次展现实力——是扮猪吃老虎被看扁后逆袭？还是在危急时刻被迫出手？
>
> **★ 代入感**：之前被轻视、被欺负的压抑感铺垫够了吗？
>
> **★ 期待感/钩子**：这章结尾留什么——暴露了实力引来更强敌人？还是被某个大人物注意到了？
>
> 另外，上一章的结尾钩子是______（回顾），这一章要衔接上。
>
> 你跟我说一下这几个方向，我就可以开始写了。"

### 示例 2：用户问写作技巧

> **用户**：我的对话写得很生硬，怎么办？
>
> **Claude** 应该这样回应：
>
> "对话生硬通常有这几个原因，你看看是哪个：
>
> 1. **标签重复**：太多'他说''她说' → 用行动代替（『他把烟按灭在桌上，『你确定？』』）
> 2. **太直接**：没有潜台词 → 让人物嘴上说的和心里想的不一样
> 3. **太正式**：现实中没人那样说话 → 加语气词、断句、口癖
>
> 要不要你发一段你写的对话，我帮你现场改一下看看效果？"

### 示例 3：用户给出模糊想法

> **用户**：我想写一个关于穿越的故事，还没想好
>
> **Claude** 应该这样回应：
>
> "穿越是个好题材！我们先从三要素出发理一下思路：
>
> **★ 爽点**：穿越后靠现代知识/前世记忆碾压？还是获得金手指逆袭？
>
> **★ 代入感**：主角穿越前是什么样的人？读者为什么关心他？
>
> **★ 期待感**：最大的悬念是什么？——他还能回到现代吗？他的真实身份是什么？
>
> 你更倾向哪个方向？（玄幻穿越/历史穿越/都市重生）有没有什么特别想写的设定？"

## 示例对话速查

### 续写时的开场
> "我读了你前面的章节。目前故事进展到 [核心冲突]，主角刚刚 [上一个事件]。
>
> 下一章你希望往哪个方向走？
> A. 继续推进主线（主角主动行动）
> B. 插入支线/配角视角
> C. 暗线揭示/埋伏笔
> D. 你来决定"

### 编辑修改时的开场
> "我读完了你的初稿，总体感觉 [一句话评价]。
>
> 优点：[2 点]
> 可以改进：[2 点]
>
> 你希望我从哪个层面开始？
> 1. 大修（结构、情节逻辑、人物弧光）
> 2. 中修（章节钩子、爽点分布、视角）
> 3. 小修（文字、对话、节奏）"

### 三要素失效时的诊断

| 症状 | 可能原因 | 对策 |
|------|----------|------|
| 读者说"太平淡" | 爽点不够或间距太大 | 缩短爽点间隔，增加小爽点 |
| 读者说"看不下去" | 期待感不足（钩子太弱） | 检查每章结尾钩子，加强悬念 |
| 读者说"不关心主角" | 代入感没建立 | 增加心理描写，强化主角的困境 |
| 读者说"太套路" | 爽点类型单一 | 多样化爽点类型（打脸/情感/揭示/力量） |

## 核心能力总览

此 skill 提供以下核心写作能力，详细内容在各参考文件中：

### 创意与构思
- 灵感收集与主题提炼
- 世界观构建与自然展现
- 代入感导向的角色设计（不完美原则、接地气动机）
- 三要素驱动的核心创意设计
- 参见 `references/creative-process.md`

### 人物鲜活度（v3.0）
- 当下体验写作法——从"发生了什么"到"正在经历什么"
- 身体是情绪的仪表盘——通过身体信号写情绪
- 意图与行动的裂缝——想做和做到了之间的戏剧性
- 内心矛盾与拉扯——两个真实但冲突的欲望
- 微观行为词典——让角色不自觉"泄露"真实感受
- 未说出口的才是真实的——留白与潜台词
- 参见 `references/character-vitality.md`

### 沉浸式叙事（v3.0）
- 叙事距离光谱——从"概括"到"当下体验"的四种层次
- 拉近叙事距离的五种方法
- 时间质地控制——紧张时放慢，平淡时加快
- 信息滴灌与隐藏——说得越少，读者感受越多
- 场景存在感——让读者相信"这个人真的在这里"
- 参见 `references/immersive-narrative.md`

### 大纲与结构
- 三线并行结构（主线、支线、暗线）
- 期待感三引擎：悬念、目标、危机
- 爽点分布与期待感曲线规划
- 章节结尾钩子设计
- 参见 `references/plot-structure.md`

### 场景构建与互动密度（v3.1）
- 行动-反应链——互动的本质是连续链条
- 三回合原则——有意义的互动至少展开3个来回
- 场景密度控制——三维度判断何时厚写、何时薄写
- 连续互动三层结构——台词+行动+潜台词每轮都有
- 场景节奏——开场→发展→转折→收尾四段结构
- 参见 `references/scene-craft.md`

### 写作技法

- 人物心理描写技法（核心重点）
- 有灵性的对话设计
- 画面感五技

- 参见 `references/writing-craft.md`
